# Implementation Plan: Secure Hidden API Keys

## Goal Description
The user requested that we fulfill the newly added QA Acceptance Tests which mandate that all API keys generated by the application are visually masked by default (e.g., `sk_proxy_****************`) to protect them from exposure during screen recordings or screen sharing sessions. Users should have explicit options to temporarily reveal the key or copy it directly to their clipboard.

## Proposed Changes

### Dashboard Key Rendering
---
#### [MODIFY] [KeyControls.tsx](file:///home/kyesh/GitRepos/googleapis_fine_grain_access_control/src/app/dashboard/KeyControls.tsx)
- Extract the raw API key rendering logic (`<code className="...">{k.key}</code>`) into a new local client component `<SecretKeyDisplay apiKey={k.key} />` or implement the inline React hook state.
- **State**: Add a `[isVisible, setIsVisible]` boolean state initialized to `false`.
- **Masking Logic**: When `isVisible` is false, render a masked version of the key: `sk_proxy_` followed by static bullet characters (e.g., `••••••••••••••••••••••••••••`).
- **Icons (lucide-react)**: 
  - Add an `Eye` / `EyeOff` button to toggle the `isVisible` state.
  - Add a `Copy` / `Check` button that uses `navigator.clipboard.writeText(apiKey)` to copy the real key to the clipboard regardless of the visibility state.

### Setup Page Documentation
---
#### [MODIFY] [page.tsx](file:///home/kyesh/GitRepos/googleapis_fine_grain_access_control/src/app/setup/page.tsx)
- Update the static placeholder string in the "Generate Key" instruction block from `sk_proxy_b02a5bf42e...` to use wildcard masking `sk_proxy_****************` to reinforce the secure default paradigm across the application documentation.

## Verification Plan
### Automated Tests
- Run `npm run build` and `npm run lint` to ensure the new state hooks and `lucide-react` imports compile cleanly under the strict Vercel configuration.

### Manual Verification
- Launch the browser subagent against the local development environment (`http://localhost:3000/dashboard`).
- Interactively create a new key.
- Assert that the newly created key is visually masked on the screen immediately upon creation.
- Subagent clicks the "Reveal" (Eye) icon and takes a screenshot verifying the raw text is exposed.
- Subagent clicks "Copy" and verifies a visual confirmation (e.g. icon change) occurs.
